A surface needs to hold pixels (Rgba's) and its width and height. The width and height will be used in rendering to images and calculating clipping, and the pixels will be the things that actually are operated on by stroke or paint operations. See the test_image_surface_with_operator test case below for an example of what that might look like.

ImageSurface provides iter(), into_iter(), and iter_mut() so that when a Cairus context calls paint, it can simply iterate through the pixels in the image surface and use a image compositing operator to operate on them. See operators.rs for those operations. A valid ImageSurface needs tangible height and width dimensions to be successfully created.
